# Automobile Data Wrangling and Analysis

## Overview

This project focuses on **data wrangling and initial exploration of an automobile dataset**. The dataset contains various characteristics of cars such as engine specifications, body style, fuel type, horsepower, and price.

The objective of this project is to **clean and prepare raw automobile data for analysis**. Data wrangling is an essential step in the data science workflow because real-world data often contains missing values, inconsistent formats, and other issues that must be addressed before analysis.

---

# Dataset

The dataset used in this project is the **Automobile Dataset** from the **UCI Machine Learning Repository**.

This dataset contains **205 observations and 26 attributes** describing different features of automobiles.

### Data Source

UCI Machine Learning Repository
[https://archive.ics.uci.edu/ml/machine-learning-databases/autos/imports-85.data](https://archive.ics.uci.edu/ml/machine-learning-databases/autos/imports-85.data)

---

# Dataset Features

Some of the key variables in the dataset include:

* **symboling** – Insurance risk rating of the vehicle
* **normalized_losses** – Relative average loss payment per insured vehicle
* **make** – Manufacturer of the car
* **fuel_type** – Type of fuel used (gas or diesel)
* **aspiration** – Engine aspiration type (standard or turbo)
* **num_doors** – Number of doors
* **body_style** – Type of car body (sedan, hatchback, convertible, etc.)
* **drive_wheels** – Type of drive system (front-wheel, rear-wheel, or four-wheel drive)
* **engine_size** – Size of the engine
* **horsepower** – Power output of the engine
* **city_mpg** – Fuel efficiency in city driving
* **highway_mpg** – Fuel efficiency on highways
* **price** – Price of the automobile
